我现在有一个用于训练的数据集（绝对路径：/data/hwh_data_folder/dataset/train）：

1. 数据：包含7500条iq采样数据，表示在2400 MHz到2500 MHz范围内的某一区间段采集的信号。
（1）采样率：数据的采样率可为5 Ms/s、20 Ms/s、30 Ms/s、40 Ms/s、50 Ms/s和80 Ms/s. 采样率与观测频宽相等；
（2）时长：数据文件的时长可为20 ms、40 ms、60 ms、80 ms、100 ms和150 ms；
（3）信号数量：一条数据内包含多个信号（1、2、3、4、6、8个信号），观测区间能够涵盖信号完整的频率成分；

注：信号个数小于4个时，文件时长为20 ms；
注：这些信号可能在时域和频域上存在重叠，也就是说存在信号之间的干扰。

（4）信号种类：信号包括Wi-Fi、BLE、ZigBee、LoRa、单载波、FM、AM以及其不同制式的类型共14种。部分信号的频宽很窄，准确识别较为困难，需要注意。
（5）存储格式：每个数据文件为16位浮点数格式存储的I（实部）与Q（虚部）交错序列，存储格式为.bin文件（0.bin、1.bin、...、7499.bin），示例读取代码：
import numpy as np
file_path = 'path/to/your/file.bin'
signal = np.fromfile(file_path, dtype=np.float16)
signal = signal[::2] + 1j * signal[1::2]

2. 标签：包含7500条标签，与数据文件一一对应。
（1）存储格式：每个标签文件为一个.json文件，对应0.bin数据文件的标签文件名为0.json；
（2）标签内容：

（2.1）对应数据文件的观测频率区间（即 observation_range）
（2.2）对应数据文件包含的每个信号的占用频段、占用时段、信号类型（即 signals 下的 start_frequency/end_frequency、start_time/end_time、class）

注：涉及频率的字段均为浮点数，单位为MHz；涉及时间的字段均为浮点数，单位为ms；信号类型取值为0到13的整数。

注：因为数据文件的采样率等于观测频宽，所以可以由标签文件中的观测频率区间知道数据文件的采样率；已知数据文件的采样率以及读取数据文件后知道它是多少个点，那么就可以由 观测时长=点数/采样率 得到。
